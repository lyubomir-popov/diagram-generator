# Product Roadmap

## Purpose

Turn this repo into a reliable batch-redesign workspace for on-brand diagrams: fast intake, repeatable SVG outputs, and a stable cold-start process for future agents.

## Stages

### Stage 1 — Workflow bootstrap ✅

Baseline-foundry-style cold-start structure copied over: inbox, handoff, active plan, roadmap, archive, and a resume agent.

### Stage 2 — Style-system baseline ✅

Local font, local icon set, draw.io scale reference, connector behavior, and compact SVG conventions documented from the first exemplar.

### Stage 3 — Batch redesign throughput (in progress)

Process the incoming diagram queue against the documented playbook, keeping outputs stylistically consistent and easy to compare.

### Stage 4 — Reusable primitives

If the next few diagrams share enough structure, extract a reusable SVG starter with shared defs, arrow shapes, and icon symbol conventions.

### Stage 5 — Optional automation

Add lightweight checks or generation helpers only if they reduce repetition without making the outputs less editable.

## Long-term direction

- Keep the repo minimal and task-focused.
- Prefer one shared documented style system over per-diagram improvisation.
- Use completed diagrams as exemplars for future cold starts.
