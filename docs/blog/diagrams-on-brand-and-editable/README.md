# Diagrams, on brand and editable

Working folder for the public communications around the diagram-generator project.

## Layout

- `blog-post/` — the ~1500-word long-form post for `ubuntu.com/blog` (brand section).
  - `outline.md` — structure and verifiable claims.
  - `long-post.md` — the draft post (first person).
  - `assets/` — hero image, before/after pairs, screenshots.
- `short-update/` — the ~100-word internal stakeholder update (first person).
- `lightning-talk/` — slides outline / speaker notes that mention this tool.
  - `assets/` — slide images and screenshots.

## Conventions

- All three deliverables stay first person and consistent on factual claims.
- "Before" images come from `diagrams/1.input/`, "after" from `diagrams/2.output/svg/` and `diagrams/2.output/draw.io/`.
- Reference real artifacts (scripts, library file, generated outputs) by repo-relative path so claims stay verifiable.
